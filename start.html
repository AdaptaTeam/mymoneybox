Manual de integração a API

Última atualização: 25/04/2019

Visão Geral & Objetivo

O objetivo desta documentação é orientar o desenvolvedor sobre como integrar com a API (NOME DA API), descrevendo as funcionalidades, os métodos a serem utilizados, listando informações a serem enviadas e recebidas, e provendo exemplos. O mecanismo de integração com (Nome da API) é simples, de modo que apenas conhecimentos intermediários em linguagem de programação para Web, requisições HTTP/HTTPS e manipulação de arquivos JSON, são necessários para implantar a solução (Nome da api) com sucesso. Nesse manual você encontrará a referência sobre todas as operações disponíveis na API REST da API (Nome da API). Estas operações devem ser executadas utilizando um Token nos respectivos endpoints dos ambientes:

Instalação

Por questão de segurança, as requisições a API precisam SEMPRE de um token válido e único em todas as requisições. Para solicitar seu primeiro token, execute a requisição "AUTH".

Para executar uma operação, combine a URL base do ambiente com a URL da operação desejada e envie utilizando o verbo HTTP conforme descrito na operação.

SANDBOX PRODUÇÃO
http://adi.adaptaconsultoria.com.br:teste/itc/api/ http://adi.adaptaconsultoria.com.br:8202/itc/api

Exemplos de requisições do tipo POST

POST

/auth/

Requisição AUTH - Privada

A requisição do tipo POST para o serviço de autenticação Privada (Auth) serve para criar uma autenticação via TOKEN para outras requisições do tipo PRIVADA. Após enviar essa requisição utilizando os parâmetros abaixo, outras requisições da aplicação serão permitidas para o TOKEN em questão, que será gerado e retornado após a requisição ser executada.

ATENÇÃO!!! Esse TOKEN será válido por até 15 minutos e somente poderá se utilizado uma ÚNICA vez, pois a cada requisição será gerado um novo TOKEN.

Requisição POST de exemplo
{
	"appToken": "CDD3D0B1C293E96933F2311DF45E52446826C1FF3C7EEBD71076761BA5E9FF25",
	"appPassword": "0253BBFC56DC68DB82AEE56127676B60B3F7F5DE2AF72D0FC974673A16313158",
	"login": "heitorlucasferreira-99@delboux.com.br",
	"userPassword": "123"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Identificação da aplicação
appPassword ALFANUMÉRICO 255 SIM Senha da aplicação
login ALFANUMÉRICO 255 SIM Nome do Usuário cadastrado na aplicação
userPassword ALFANUMÉRICO 255 SIM Senha do Usuário cadastrado na aplicação
Retorno da Requisição AUTH - Privada
{
    "token": "FA9622016767FE46EA6601042586C5CD6D4AFB84723105E353DDDE07A55A8461",
    "user": {
        "firstname": null,
        "lastname": null,
        "name": "Vinicius Yago Bruno Pereira",
        "email": "heitorlucasferreira-99@delboux.com.br",
        "login": "heitorlucasferreira-99@delboux.com.br",
        "company": 1,
        "isactive": true,
        "phone": "",
        "role": "CUSTOMER"
    },
    "hasError": false,
    "error": null
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Valor variável e retornado a cada requisição
firstname ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
lastname ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
name ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
email ALFANUMÉRICO 255 SIM E-mail do cadastro
login ALFANUMÉRICO 255 SIM Se não for informado no cadastro, retornará o E-mail
company NUMÉRICO 50 SIM Código de identificação da empresa
isactive TEXTO 5 SIM Informativo sobre o Status da conta.
phone NUMÉRICO 50 NÃO Será retornado se existir no cadastro
role TEXTO 80 SIM Retornará o tipo do perfil do cadastro
haserror TEXTO 5 SIM Retornará "TRUE" se houver erros na requisição
error ALFANUMÉRICO 255 SIM Se houver erros, retornarão nesta chamada
